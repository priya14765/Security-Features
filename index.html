What is OS Security and Why Does it Matter?

🛡 Crucial Protection

OS security is the bedrock of digital safety. It dictates how hardware and software interact, ensuring malicious actors cannot compromise the core integrity of the machine.

⚙ Core Functions

  • Safeguards sensitive information
  • Prevents unauthorized access
  • Ensures system stability
  • Mitigates cyber threats

AUTHENTICATION VS AUTHORIZATION

Authentication

"Who are you?"

  • • Verifies identity (passwords, bio)
  • • Occurs at initial login
  • • Controls system access

Authorization

"What can you do?"

  • • Determines permissions
  • • Occurs after authentication
  • • Enforces "Least Privilege"

ACCESS CONTROL MODELS

DAC (Discretionary)

Owner decides who has access (e.g., Windows file sharing).

👥

MAC (Mandatory)

System-wide policy based on clearance levels (SELinux, Military).

🎖

RBAC (Role-Based)

Access assigned to roles, not individuals ("Admin", "User", "Guest").

👔

MEMORY PROTECTION

PREVENTING SYSTEM CRASHES AND CODE INJECTION

🚧

Isolation

Separating process memory so one crash doesn't take down the OS.

🎲

ASLR

Randomizing memory layout to prevent predictable exploit targeting.

🚫

DEP / NX

Marking memory as non-executable to stop shellcode injection.

SANDBOXING IN OS SECURITY

What is a Sandbox?

An isolated execution space with restricted access to system resources.

  • ✓ Restricts file system access
  • ✓ Limits network capabilities
  • ✓ Controls hardware device access
  • ✓ Isolates from other processes

Why Use Sandboxing?

Containment prevents compromised apps from damaging the system.

  • 🛡 Contains exploits
  • 🔒 Minimal privilege isolation
  • 👾 Limits lateral movement
  • 🧪 Safe code testing

Browser Sandboxing

Tabs run isolated from OS.

Mobile OS

iOS/Android sandbox each app.

Antivirus

Suspected files in isolated VMs.

THE THREAT LANDSCAPE

OS-Centric Malware Types

Rootkits

Hide existence. Kernel-mode rootkits modify OS system calls to become invisible.

Ransomware

Encrypts files via OS APIs, deletes shadow copies to prevent recovery.

Spyware

Hooks into OS input interrupts to steal credentials and keystrokes.

VIRTUALIZATION & CONTAINERS

Hypervisor Security

The Hypervisor (VMM) manages virtual machines. If compromised, an attacker can escape the guest OS (VM Escape).

  • Type 1 (Bare Metal): Higher security.
  • Type 2 (Hosted): Larger attack surface.

Container Isolation

Containers share the host OS kernel. A kernel exploit can compromise the entire host system.

  • Use Namespaces & Cgroups.
  • Run containers as non-root.

CRITICAL CASE STUDIES

Real-world OS security failures and recoveries.

CASE 1: WANNACRY (2017)

⚠ ROOT CAUSE

Exploited "EternalBlue" in SMBv1 (MS17-010). Remote code execution without authentication on unpatched Windows.

✅ THE FIX

Apply MS17-010 patch. Disable SMBv1. Network segmentation to prevent lateral movement.

CASE 2: SOLARWINDS (2020)

⚠ ROOT CAUSE

Compromised build system injected malicious code into legitimate updates, signed with stolen certificate.

✅ THE FIX

Software Bill of Materials (SBOM). Enhanced DNS monitoring. Zero-trust architectures.

CASE 3: MELTDOWN & SPECTRE (2018)

⚠ ROOT CAUSE

Hardware flaw in speculative execution. OS memory isolation failed to prevent reading kernel memory from user space.

✅ THE FIX

Kernel Page Table Isolation (KPTI). CPU microcode updates. Browser timer restrictions.

CASE 4: STUXNET (2010)

⚠ ROOT CAUSE

Spread via infected USB drives using four Windows zero-day exploits to reach air-gapped SCADA systems.

✅ THE FIX

USB control policies. Application whitelisting. Air-gap verification. Patch management for legacy systems.

CASE 5: DIRTY COW (2016)

⚠ ROOT CAUSE

Race condition in Linux Kernel Copy-on-Write (CVE-2016-5195). Local user gained root access.

✅ THE FIX

Immediate kernel patch. Kernel Live Patching (Ksplice) for fixes without rebooting.
